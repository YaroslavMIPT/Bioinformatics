# Отчет по домашнему заданию №5: Контроль качества и тримминг ридов

## 1. Контроль качества исходных данных
Для первичного анализа сырых данных использовались утилиты FastQC и MultiQC. Анализ показал следующие результаты:

* **Качество по длинам (Per base sequence quality):** Наблюдалось заметное падение качества (ниже Q20) на концах ридов, начиная примерно со 120-130 позиции.
* **Наличие адаптеров:** В библиотеках обнаружено присутствие стандартных адаптеров Illumina (Universal Adapter).
* **Длина ридов:** Исходные данные представлены ридами фиксированной длины 150 bp.
* **Содержание GC:** Распределение GC-состава соответствует ожидаемому для биологического образца, без резких аномальных пиков.

## 2. Тримминг данных (Предобработка)
Для очистки данных была выбрана утилита **fastp**, так как она выполняет тримминг быстрее аналогов и сразу генерирует наглядные отчеты.

**Примененные параметры:**
* Автоматический поиск и удаление адаптеров.
* Обрезка концов ридов с использованием скользящего окна (качество ниже 20).
* Удаление ридов, длина которых после тримминга стала меньше 30 bp.

Скрипт запуска находится в репозитории по пути: `scripts/2_trimming.sh`

## 3. Анализ данных после тримминга
После обработки инструментом fastp и повторного запуска контроля качества зафиксированы следующие улучшения:

* **Adapter Content:** Содержание адаптеров снизилось до нулевых значений.
* **Per base sequence quality:** Среднее значение качества по всей длине рида теперь находится в «зеленой зоне» (Q > 30). Низкокачественные «хвосты» были успешно отрезаны.
* **Sequence Length Distribution:** В отчете появилось распределение длин ридов. Большинство ридов сохранили длину 140-150 bp, однако появились и более короткие фрагменты, что подтверждает работу алгоритма обрезки по качеству.
* **Статистика фильтрации:** Более 95% ридов успешно прошли фильтрацию, что говорит о высоком качестве исходной библиотеки и корректности выбранных параметров тримминга.

## Вывод
В ходе работы данные были успешно очищены от технических последовательностей и низкокачественных участков. Полученные файлы `_trimmed.fastq.gz` полностью готовы для дальнейшей сборки генома (de novo assembly) или картирования на референс.
